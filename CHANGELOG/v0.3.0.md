# Release v0.3.0

## Highlights

### 🚀 Major Features

**Catalog Architecture**
- Complete application lifecycle management system with REST APIs, web UI, and CLI integration
- Deploy, monitor, and manage AI service applications through unified interface
- JWT-based authentication and authorization
- Pod/container health status tracking and monitoring
- PostgreSQL backend for application metadata storage

**Repository Code Reorganization**
- Restructured codebase into services/, ui/, and images/ directories for better modularity
- Clear separation between backend microservices, frontend applications, and container images
- Renamed rag-base to service-base for consistency
- Migration: spyre-rag/src/ → services/, digitize-ui/ → ui/digitize/, catalog-ui/ → ui/catalog/

**Conversational RAG**
- Multi-turn conversation support with automatic query rephrasing
- Client-side session management with stateless backend
- Conversation history truncation (last 10 messages)
- Pronoun resolution and context injection
- Multi-language support (English, German, Italian, French)

**Similarity Search Service**
- New standalone similarity search API with configurable modes
- Dense (k-NN), sparse (BM25), and hybrid search support
- Optional reranking with Cohere
- REST API endpoint: POST /v1/similarity-search
- Performance timing information in response headers

**HTTPS/TLS Support**
- Automatic HTTPS enablement with Caddy reverse proxy integration
- Self-signed certificate generation for development
- Custom certificate support for production
- Dynamic route registration via Caddy Admin API
- HTTP/2 and HTTP/3 support

**PostgreSQL Database Integration**
- Migration from JSON file storage to PostgreSQL for digitize and summarize services
- ACID guarantees and concurrent write performance
- Efficient querying with built-in indexing
- Database schema with documents and jobs tables
- Backup/restore capabilities

**Enhanced Summarization Service**
- Asynchronous job-based summarization with status tracking
- Direct and chunked summarization strategies
- Streaming API support
- CRUD endpoints for job management (GET /v1/jobs, GET /v1/jobs/{job_id})
- DOCX file support with configurable summarization levels


### 🔧 Key Improvements

**Performance & Reliability**
- Asynchronous processing for document ingestion and summarization
- Database connection pooling for efficient resource management
- Streaming APIs for summarization responses
- Orphan job recovery mechanism for summarize service
- Retry mechanism for similarity service OpenSearch connections
- Performance timing headers for similarity API

**Backend Enhancements**
- Import/Export APIs for metadata backup and restore
- Network-based data transfer without file system access
- Batch processing support for import operations
- Enhanced error handling with proper HTTP status codes
- Improved logging with request IDs and structured JSON logging
- Health endpoints with liveness and readiness probes

**Security & Compliance**
- vLLM API key-based authentication (optional, disabled by default)
- Non-root user support for all services with arbitrary UID compatibility
- Automatic password generation for OpenSearch and PostgreSQL
- Kubernetes Secret integration for sensitive data
- Registry authentication for private image pulls
- SELinux policy integration for Podman

**Developer Experience**
- Comprehensive unit test coverage for all services (chatbot, digitize, summarize, common)
- CI/CD enhancements with automated testing and version bumping
- Extensive proposal documents for major features
- Updated Swagger API documentation
- AI Services Podman development guide
- Improved CLI help messages and error feedback

**Multi-Language Support**
- German language support for chatbot conversational RAG and digitize table summarization
- Italian and French language support for chatbot
- Language-specific prompt templates
- Automatic language detection with Lingua library
- Configurable role labels and stop sequences per language

**New Container Images**
- PostgreSQL 16 UBI-based image with non-root support
- LiteLLM proxy image for multi-provider LLM integration with WatsonX support
- Caddy reverse proxy image for HTTPS termination
- Updated service-base image with Docling v2.95.0

### ⚠️ Breaking Changes

**Code Organization**
- Python import paths updated for new directory structure (spyre-rag/src/ → services/)
- Image name changed from rag-base to service-base
- File paths for services moved to new locations

**Deployment Architecture**
- New catalog-based deployment model available (legacy mode still supported with --legacy flag)
- PostgreSQL required for digitize and summarize services (JSON storage deprecated)
- New environment variables and CLI flags for catalog configuration
- Caddy proxy integration for HTTPS (optional but recommended)

**Migration Required**
- Update import statements in custom code for new directory structure
- Migrate JSON metadata to PostgreSQL using import API
- Update deployment scripts for new catalog architecture
- Configure HTTPS with Caddy for production deployments

### 📝 Upgrade Notes

**Prerequisites**
- PostgreSQL 16+ (for digitize and summarize services)
- Caddy 2.x (for HTTPS support, optional)
- Updated CLI binary to latest version

**Migration Steps**
1. Backup existing JSON metadata files
2. Update CLI binary to v0.3.0
3. Configure catalog (optional): `ai-services catalog configure`
4. Deploy with new architecture: `ai-services application create`
5. Import existing metadata using `/v1/import` API endpoint

**Compatibility**
- Existing v0.2.0 deployments continue to work
- JSON file storage still supported (deprecated, use PostgreSQL)
- Old CLI commands remain functional
- New catalog features require updated CLI and catalog backend
- Legacy deployment mode available with --legacy flag
