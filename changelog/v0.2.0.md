# Release v0.2.0
Date: 2026-03-07

## Summary
This release includes 39 new feature(s), 35 bug fix(es), 14 improvement(s). New features include: New template for Summarization (#405) Key fixes: AIS-706-fix broken tests (#418).

## Highlights and Major Changes

### 🎯 Major Features

#### Catalog Service & UI
- **Catalog UI Implementation**: Complete catalog UI with React and Carbon Design System, including login/logout functionality, authentication APIs, and application management pages
- **Catalog API Server**: Full implementation with authentication (login, logout, whoami commands) and backend integration
- **Profile Management**: Added profile panel and logout dialog box with left navigation pane

#### OpenShift Support
- **Multi-Runtime Architecture**: Introduced runtime interface supporting both Podman and OpenShift platforms
- **OpenShift Bootstrap**: Complete bootstrap manifests and configuration for OpenShift deployment
- **Helm Charts**: RAG dev application deployment via Helm charts for OpenShift
- **Application Management**: Full lifecycle support (create, delete, ps, logs, info, start, stop) for OpenShift runtime

#### New Services & APIs
- **Summarization Endpoint**: New summarization API service with dedicated templates and design documentation
- **Digitize API**: Document digitization service skeleton with endpoint design and integration into RAG configurations
- **Asynchronous Ingestion**: New asynchronous workflow for document ingestion

#### Enhanced RAG Pipeline
- **New Ingestion Pipeline**: Complete overhaul with improved PDF processing, validation, and logging
- **OpenSearch Integration**: Replaced Milvus with OpenSearch as vector database
- **Performance Monitoring**: Introduced performance data collection and retrieval utilities
- **Request Traceability**: Added UUID-based request tracking

### 🔧 Infrastructure & Architecture

#### Code Quality & Testing
- **E2E Testing Framework**: Comprehensive end-to-end testing suite with XML reporting and golden dataset validation
- **Nightly Test Runs**: Automated nightly testing infrastructure
- **Linter Fixes**: Extensive code quality improvements (100+ linter fixes across godot, godox, nestif, funlen, cyclop, etc.)

#### Refactoring & Organization
- **Dependency Inversion**: Major refactoring moving business logic from cmd to internal/pkg
- **Asset Structure**: Reorganized to support multiple runtimes (Podman/OpenShift)
- **Runtime Abstraction**: Introduced runtime interface for better multi-platform support

#### Security & Compliance
- **CVE Fixes**: Multiple high-severity CVE fixes (wheel, pdfminer.six, urllib3, circl, quic-go)
- **TLS Support**: Enabled TLS for Routes
- **Security Hardening**: Disabled automountServiceAccountToken for services not requiring cluster access
- **License Scanning**: Added license scan for ai-services images

### 🚀 Operational Improvements

#### Deployment & Configuration
- **RHOAI Assets**: Added Red Hat OpenShift AI assets and configuration changes
- **Image Management**: Optimized image pulling (only download missing images), multi-arch tools image
- **Storage Validation**: Added storage class and node label validation for OpenShift
- **Spyre Integration**: Spyre cluster policy configuration and validation, device detection using device ID

#### Developer Experience
- **Flag Validation**: Enhanced flag validation based on runtime
- **Better Documentation**: Updated help sections, added design docs for new features
- **CI/CD Improvements**: GitHub Actions optimization, image name consistency checks, version bump enforcement
- **PR Preview**: Jenkins pipeline for PR preview feature

### 🐛 Notable Bug Fixes
- Fixed broken tests (AIS-706)
- Fixed ImagePullPolicy default values
- Fixed volume mount in OpenSearch StatefulSet
- Fixed bootstrap failures for soft checks
- Bug fixes for PDF processing (0 table handling, heavy PDF worker size)
- Fixed duplicate artifact names in license scan
